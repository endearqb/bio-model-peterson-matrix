02 / Core Logic Matrix Implementation

Peterson 矩阵的代码化

数学结构天然适合 numpy 矩阵运算：

dC/dt = 流量项 + νᵀ · ρ
# 1. 计算所有过程速率 ρ (8 维向量) rho = calc_rates(C, params) # 2. 矩阵乘法：化学计量矩阵转置 νᵀ (13x8) 与速率 ρ (8x1) 相乘 # 直接得到 13 个组分的生物反应变化率！ reaction = nu.T @ rho

优雅之处：无需手写 13 个冗长的独立微分方程。

构建化学计量矩阵 (ν)

将第 6 章的表格精确翻译为代码（行=过程，列=组分）：

def build_stoichiometry(p): nu = np.zeros((N_PROC, N_COMP)) # 8x13 矩阵 # ── 过程 1：异养菌好氧生长 ── nu[0, S_S] = -1 / p['Y_H'] # 消耗底物 nu[0, X_BH] = +1 # 生成生物量 nu[0, S_O] = -(1 - p['Y_H']) / p['Y_H'] # 耗氧 nu[0, S_NH] = -p['i_XB'] # 吸收氨氮合成细胞 # ── 过程 2：异养菌缺氧生长 (反硝化) ── # ... return nu
05 / Reactor CSTR ODE System

CSTR 控制方程

组装对流、反应、排泥和曝气项，供 scipy.integrate.solve_ivp 调用：

def cstr_ode(t, C, C_in, HRT, SRT, KLa, SO_sat, params, nu): C = np.maximum(C, 0.0) # 防止数值误差导致负浓度 rho = calc_rates(C, params) reaction = nu.T @ rho # 生物反应项 convection = (C_in - C) / HRT # 进出水对流项 sludge_loss = np.zeros(N_COMP) for i in PARTICULATE: # 仅颗粒态受排泥影响 sludge_loss[i] = -C[i] / SRT aeration = np.zeros(N_COMP) aeration[S_O] = KLa * (SO_sat - C[S_O]) # 氧传递项 return convection + reaction + sludge_loss + aeration
06 / Execution Running the Simulation

运行与可视化

使用 LSODA 算法处理刚性 (Stiff) ODE 系统：

sol = solve_ivp( fun=cstr_ode, t_span=(0, 60), # 模拟 60 天 y0=C0, # 初始状态 args=(...), method='LSODA', # 关键：应对刚性系统 rtol=1e-6, atol=1e-8 )

典型稳态结果 (全好氧)：

  • S_S (底物): 70 → < 2 g/m³
  • S_NH (氨氮): 35 → < 2 g/m³
  • S_NO (硝氮): 0 → ~28 g/m³ (硝化产物)
  • X_BH (异养菌): ~1700 g/m³

09 / Debugging Numerical Issues

常见数值问题调试

求解极慢 / 报警 (刚性 ODE)

原因：微生物生长极快与停留时间尺度相差大。
解决：必须在 solve_ivp 中使用 method='LSODA' 或 'Radau'。

组分浓度出现负值

原因：求解器步长过大导致越界，引发后续报错。
解决：在 ODE 函数开头及速率计算时使用 C = np.maximum(C, 0.0) 截断。

除以零错误 (ZeroDivisionError)

原因：水解过程中的 X_S / X_BH 项在初始生物量为 0 时报错。
解决：对除数项设置极小下限，如 max(C[X_BH], 1e-10)。
